# راه‌اندازی ابر (توسعه‌دهنده)

این راهنما برای **تو** است، نه برای مشتری.

## ۱. ساخت پروژه Firebase

1. برو به [console.firebase.google.com](https://console.firebase.google.com)
2. **Add project** بساز (مثلاً `book-for-all`)
3. Google Analytics را می‌توانی خاموش کنی

### Authentication

1. **Build → Authentication → Get started**
2. تب **Sign-in method** → **Email/Password** را Enable کن
3. تب **Users → Add user**
4. یک ایمیل و رمز برای مشتری بساز (مثلاً `admin@client.com`)
5. این ایمیل/رمز را فقط به مشتری بده

### Firestore

1. **Build → Firestore Database → Create database**
2. حالت را **production** بگذار
3. قوانین را از فایل `firestore.rules` کپی کن و Publish کن:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /books/{bookId} {
      allow read: if true;
      allow create, update, delete: if request.auth != null;
    }
  }
}
```

یا با Firebase CLI:

```bash
npm install -g firebase-tools
firebase login
firebase use YOUR_PROJECT_ID
firebase deploy --only firestore:rules
```

### Web App config

1. Project settings → Your apps → **Web** (`</>`)
2. مقادیر را در `js/firebase-config.js` بگذار (جای `YOUR_...`)

## ۲. Seed کتاب‌های اولیه

1. سایت را محلی یا آنلاین باز کن
2. برو به `#/admin` و با ایمیل/رمز ادمین وارد شو
3. اگر ابر خالی باشد، دکمه **بارگذاری کتاب‌های اولیه** را بزن
4. کتاب‌های `data/books.json` یک‌بار به Firestore منتقل می‌شوند

## ۳. دیپلوی روی Cloudflare Pages

### از داشبورد

1. کد را به GitHub push کن
2. [dash.cloudflare.com](https://dash.cloudflare.com) → **Workers & Pages → Create → Pages → Connect to Git**
3. تنظیمات:
   - **Build command:** خالی
   - **Build output directory:** `/` یا `.`
4. Deploy
5. لینک سایت را بردار (مثلاً `https://book-for-all.pages.dev`)

### با Wrangler (اختیاری)

```bash
npm install -g wrangler
npx wrangler pages deploy . --project-name=book-for-all
```

## ۴. چک نهایی

- [ ] کاربران بدون لاگین کتاب‌ها را می‌بینند
- [ ] `#/admin` با ایمیل/رمز درست وارد می‌شود
- [ ] افزودن کتاب → در مرورگر دیگر / موبایل دیگر هم دیده می‌شود
- [ ] کاربر عادی `#/add` را نمی‌بیند / دسترسی ندارد

- ایمیل ادمین
- رمز ادمین
